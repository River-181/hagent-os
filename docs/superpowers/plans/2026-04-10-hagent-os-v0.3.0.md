# Plan: HagentOS v0.3.0 — Paperclip 라이브 피드백 반영

## Context

**현재**: v0.2.0 완료 (21페이지, 40+컴포넌트, 15 API, ~7,000 LOC). 커밋 5e91989.
**피드백 소스**: 사용자의 Paperclip 라이브 인스턴스 운영 경험 (127.0.0.1:3101, 탄자니아 영어학원).
**핵심 문제**: 현재 HagentOS는 "보이기만 하는" 수준. Paperclip처럼 "실제로 작동하는" 수준으로 올려야 함.
**마감**: 2026-04-13 (D-2)

---

## 사용자 피드백 12항 (Paperclip 라이브 운영 경험)

1. **칸반 드래그**: 보드에서 카드를 옮기면 상태 변경 + "진행 중"으로 옮기면 에이전트 자동 실행
2. **사이드바 개편**: 에이전트 각각 보임, org/skills/costs는 Paperclip "Company" 섹션처럼 분리
3. **인박스 = 액션 허브**: 승인/거절/이슈 이동 인라인으로 가능해야 함
4. **프로젝트 단위**: 온보딩 프로젝트 → 하위 이슈들 → 에이전트가 처리. 코멘트→재실행
5. **에이전트 정체성**: SOUL.md + HEARTBEAT.md + 일별 메모리 (Paperclip 패턴 확인됨)
6. **에이전트 상세 디테일**: 대시보드/인스트럭션/스킬/설정/실행이력/예산 + 터미널 로그
7. **OrgChart 인터랙션**: 클릭→에이전트 이동, 상태점, 직원(인간)+에이전트 구분
8. **토글 투명 버그**: 모든 토글/스위치가 보이지 않음 (shadcn Switch 미설치)
9. **학생/학부모 관리**: DB 스키마 있으나 UI 없음. 개인정보 에이전트 접근 제한 필요
10. **지식베이스 강화**: 검색, 문서 가져오기(hwp 변환 등), 에이전트가 스킬로 문서 추가
11. **일정 캘린더**: 수업/상담/이벤트/법정기한, 수업→학생 연결, 출석/수납 (복잡하면 비활성 옵션)
12. **에이전트 아이콘**: 각 에이전트마다 커스텀 이모지/아이콘 (Paperclip: sparkles, cpu, cog, lightbulb)

### Paperclip 라이브 인스턴스에서 배운 것
- 에이전트 파일 구조: `companies/{id}/agents/{id}/instructions/` (SOUL.md, HEARTBEAT.md, AGENTS.md, TOOLS.md)
- 에이전트 메모리: `agents/{id}/memory/YYYY-MM-DD.md` (일별 자동 생성)
- 현재 운영: 원장(CEO), CTO, MarketingLead, OperationsManager, HeadOfCurriculum — 5 에이전트
- 프로젝트: "Onboarding" 프로젝트 아래 7개 이슈 (채용 태스크들)
- 스킬: paperclip, paperclip-create-agent, paperclip-create-plugin, para-memory-files

## v0.3.0 구현 계획

### 배치 A: 인터랙션 핵심 (소~중 난이도, 높은 가시성)

#### A1. 칸반 드래그 → 상태 변경 + 에이전트 자동 실행
**파일**: `ui/src/components/KanbanBoard.tsx`, `ui/src/api/cases.ts`
- HTML5 드래그앤드롭 (`draggable` + `onDragOver` + `onDrop`, 외부 라이브러리 없음)
- 드롭 시 `casesApi.update(id, { status })` PATCH 호출
- `in_progress`로 이동 시 → `agentsApi.wakeup(assigneeAgentId)` 자동 트리거
- 드래그 중 시각적 피드백 (opacity 0.5, 드롭 영역 border 하이라이트)
- TanStack Query invalidation으로 즉시 반영

#### A2. OrgChart 클릭 + 상태점 + 인간 직원
**파일**: `ui/src/pages/OrgChartPage.tsx`
- AgentNode에 `onClick={() => navigate(\`/${orgPrefix}/agents/${agent.id}\`)}`
- `cursor-pointer` + hover scale 효과
- 에이전트 아이콘 표시 (agent.icon 필드 → lucide 아이콘 매핑)
- instructors 데이터 조회 → 인간 직원 노드 (파란 배경으로 구분, "직원" 배지)

#### A3. 토글/스위치 설치 + 적용
**명령**: `cd ui && npx shadcn@latest add switch`
- `SettingsPage.tsx` — 투명 토글 → `<Switch>` 교체
- `RoutinesPage.tsx` — 루틴 활성화 토글
- `AgentDetailPage.tsx` Skills 탭 — 스킬 활성화 토글

#### A4. 에이전트 아이콘 시스템
**DB**: `agents.icon` 컬럼 (text, nullable) — 이미 존재 여부 확인 필요
**UI**: `Identity.tsx` — icon 필드 있으면 해당 lucide 아이콘 렌더링
**매핑**: brain→오케스트레이터, shield→민원, heart→이탈, calendar→일정, sparkles→마케팅, cpu→기술, cog→운영, lightbulb→교육
**시드**: 4 에이전트에 아이콘 추가

### 배치 B: 인박스 + 프로젝트 (중 난이도)

#### B1. 인박스 실제 데이터 + 액션
**파일**: `ui/src/pages/InboxPage.tsx` 전면 재작성
- 목업 → notifications 테이블 + approvals 테이블 조회
- 승인 대기 항목: ApprovalCard 인라인 렌더 (승인/거절 버튼)
- 케이스 업데이트: "케이스 보기" → navigate
- 에이전트 완료: 결과 요약 + "상세 보기"
- 서버: `GET /api/organizations/:orgId/notifications` 라우트 추가

#### B2. 프로젝트 페이지
**파일**: `ui/src/pages/ProjectsPage.tsx` (신규), `ui/src/pages/ProjectDetailPage.tsx` (신규)
- 프로젝트 = ops_groups 활용 (스키마 이미 존재)
- 목록: 이름, 설명, 상태, 하위 케이스 수
- 상세: 개요 + 하위 케이스 목록
- 서버: `GET/POST /organizations/:orgId/projects` 라우트 (`server/src/routes/projects.ts`)
- 시드: "온보딩" 프로젝트 + 하위 케이스 2-3개 연결

### 배치 C: 에이전트 깊이 (중 난이도, 데모 임팩트 높음)

#### C1. 에이전트 메모리/정체성 파일 시스템
**서버**: agents 테이블에 `memory` JSONB 컬럼 (또는 기존 것 활용)
- `GET /api/agents/:id/memory` → memory JSONB 반환
- `PATCH /api/agents/:id/memory` → memory 업데이트
- 시드: 각 에이전트에 SOUL(정체성 요약), 최근 실행 메모 포함

**UI**: AgentDetailPage 개요 탭 "메모리" 섹션
- 최근 메모리 엔트리 (날짜별 카드)
- 정체성 요약 표시

#### C2. 에이전트 상세 강화 — 실행 트랜스크립트
**파일**: `ui/src/pages/AgentDetailPage.tsx`
- Runs 탭: 실행 클릭 → 트랜스크립트 펼침 (input/output 구조화)
- Dashboard 탭: 최근 실행 차트
- Configuration 탭: 모델, 어댑터 타입, 환경 변수 표시
